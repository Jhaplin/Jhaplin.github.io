二、Ubuntu 源代码方式安装软件的完整流程（以 htop 为例）

htop 是 Linux 系统下的一个交互式实时进程监控工具，提供彩色界面，可显示 CPU、内存使用情况，并支持进程管理（如杀死、排序、筛选）。

1、环境准备

按照htop的项目官方文档要求, 安装相应依赖

2、源代码方式安装 htop

（1）下载源代码包（国内镜像）

使用清华 TUNA 镜像下载 htop 源码：

（2）解压源代码压缩包, 进入源代码目录

（3）执行 ./configure 生成 Makefile

注意：htop 官方文档要求 必须先运行./autogen.sh

因为htop 的源代码包没有包含已生成的 configure 文件，而是提供了 configure.ac 和 Makefile.am，因此需要先运行 autogen.sh 脚本来调用 autoconf/automake 工具，自动生成 configure 和 Makefile.in，这样才能继续执行 ./configure → make → make install 的标准编译流程。

然后生成 Makefile, 同时指定安装目录

（4）执行 make 编译

（5）执行 sudo make install 安装

安装后 htop 会被放入/usr/local/bin/htop
